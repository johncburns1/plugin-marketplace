# Technical Project Manager Skill

A principal-level skill for transforming product requirements into well-organized GitHub issues and milestones.

## What It Does

Creates small, focused GitHub issues (max 2 story points) from product requirements with:

- Consistent templates and structure
- Milestone organization
- Context from PRODUCT.md and ARCHITECTURE.md
- Iterative, incremental approach

## When to Use

Use this skill when you want to:

- Plan engineering work for a project
- Create GitHub issues from product requirements
- Break down features into actionable tasks
- Organize work into milestones

## Prerequisites

- `gh` CLI installed and authenticated
- PRODUCT.md file in project root (recommended)
